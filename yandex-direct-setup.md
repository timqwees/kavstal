## Общая стратегия

**Бюджет:** 100% → Поиск 60% | РСЯ 25% | Наружная (DOOH) 10% | Умный баннер 5%
**Автостратегия везде:** «Максимум конверсий» с таргет CPA (целевая стоимость конверсии).
**Гео:** Москва + МКАД + 30 км МО (основное). Отдельные кампании для областей — только если будет поток.
**Типы соответствия:** 80% широкое + 20% фразовое. Точное вхождение — только для минус-слов.
**Сбор ключей:** Яндекс.Wordstat + Keys.so (снятие у конкурентов) + авторасширение Директа.
**Общий объём:** 25 000–35 000 ключей на старте, далее расширение до 50 000+ за счёт авторасширения.

3. **Креативы РСЯ:** CTR < 0.3% → замена креатива. Обновление минимум раз в 2 недели.

## Ежемесячно
4. **Ключи:** досбор по неэффективным группам. Keys.so — свежие ключи конкурентов.
5. **Стратегии:** пересмотр CPA (если факт выше плана на 30%+ — либо снижать CPA, либо убирать группы).
6. **РСЯ аудитории:** Look-alike на 100+ конверсий (если набрали).

## Сезонность
7. **Апрель–Октябрь:** строительный сезон. Бюджет ×1.4. Включить все РСЯ аудитории.
8. **Ноябрь–Март:** низкий сезон. Бюджет ×0.7. Упор на газгольдеры, метизы, нержавейку (ремонты внутри помещений).

# ЧАСТЬ 4: ОТСЛЕЖИВАНИЕ КОНВЕРСИЙ (КОД НА САЙТЕ)

## Что уже внедрено в код kavstal.ru

### 1. yclid (Яндекс.Директ Click ID)
При заходе с рекламы yclid автоматически:
- Сохраняется в localStorage на 30 дней
- Добавляется как `<input type="hidden" name="yclid">` во все формы
- Передаётся в Metrika при отправке целей через `trackGoal()`

**Нужно в Директе:** Включить «Автоматические UTM-метки» в настройках кампании (вкл по умолчанию).

### 2. Цели в Метрике (создать вручную в Метрике → Цели)

| Тип цели | ID цели (data-goal) | Где срабатывает | Приоритет |
|----------|---------------------|-----------------|-----------|
| Цель — визит | `calculator_request` | Отправка формы калькулятора на главной | ★★★ |
| Цель — визит | `callback` | Форма обратного звонка (футер) | ★★★ |
| Цель — визит | `open_callback` | Клик «Перезвоните мне» | ★★ |
| Цель — визит | `click_phone` | Клик по любому номеру телефона на сайте | ★★★ |
| Цель — визит | `buy_one_click` | Клик «Купить в 1 клик» на карточке товара | ★★ |
| Цель — визит | `buy_one_click_form` | Отправка формы «Купить в 1 клик» | ★★★ |
| Цель — визит | `checkout` | Отправка формы оформления заказа | ★★★ |
| Цель — визит | `click_email` | Клик по email | ★ |

### 3. E-commerce цели (dataLayer → Метрика автоматически)

| Событие dataLayer | Где срабатывает | Что передаёт |
|-------------------|-----------------|--------------|
| `view_item` | Каждая страница товара | ID товара, название, цена |
| `add_to_cart` | Клик «В корзину» (десктоп + мобильный) | ID, название, цена, количество |
| `purchase` | Страница успешного заказа `/order/*/success` | ID заказа, сумма, валюта, состав заказа |

**В Метрике эти события НЕ НУЖНО создавать как цели вручную** — Яндекс.Метрика автоматически считывает их из dataLayer (параметр `ecommerce:'dataLayer'` при инициализации уже установлен).

**В Директе:** Импортировать E-commerce цели из Метрики или создать цели со ссылкой на события dataLayer.

### 4. Какие цели создать в Яндекс.Директе

Рекомендуемые цели для стратегии «Максимум конверсий»:

| Цель в Директе | Источник | Вес |
|----------------|----------|-----|
| `checkout` — оформление заказа | Метрика (цель checkout) | 1.0 |
| `buy_one_click_form` — заявка с товара | Метрика (цель buy_one_click_form) | 0.9 |
| `callback` — обратный звонок | Метрика (цель callback) | 0.8 |
| `calculator_request` — заявка с калькулятора | Метрика (цель calculator_request) | 0.7 |
| `click_phone` — звонок с сайта | Метрика (цель click_phone) | 0.6 |
| `purchase` — оплаченный заказ (dataLayer) | Метрика (e-commerce) | 1.0 |
| `add_to_cart` — добавление в корзину (dataLayer) | Метрика (e-commerce) | 0.3 |

В стратегии «Максимум конверсий» указать `checkout` + `buy_one_click_form` + `callback` как основные конверсии. Остальные — вспомогательные (только для статистики).

### 5. Офлайн-конверсии

Если клиент оплачивает или подтверждает заказ после звонка (не на сайте):
1. yclid сохраняется при заходе с рекламы и приходит с формой
2. Выгружаете из CRM: номер телефона + yclid + сумма + дата
3. Загружаете через API Яндекс.Директа (офлайн-конверсии) или через CSV в интерфейсе
4. Яндекс свяжет конверсию с конкретным кликом и покажет реальную стоимость привлечения клиента

### 6. Проверка перед запуском

- [ ] yclid сохраняется в localStorage: открыть консоль → `localStorage.getItem('yclid')`
- [ ] Цели приходят в Метрику: режим отладки Метрики (расширение «Яндекс.Метрика»)
- [ ] dataLayer события: консоль → `dataLayer` — должны быть view_item/add_to_cart/purchase
- [ ] Все формы отправляют yclid: `document.querySelectorAll('form [name="yclid"]')`
- [ ] Телефонные номера отслеживаются: `document.querySelectorAll('a[href^="tel:"]')` — должно быть >0
